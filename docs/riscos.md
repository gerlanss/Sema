# Riscos

## Semantica inflada cedo demais

Se a linguagem tentar modelar tudo no MVP, ela vira um Frankenstein caro de manter.

## Parser permissivo demais

Se o parser aceitar porcaria demais e empurrar tudo para fases seguintes, os diagnosticos ficam fracos.

## Effects ambiguos

Se `effects` virar texto decorativo, a linguagem trai sua propria proposta.

## Geracao acoplada a framework

Se o projeto tentar agradar framework cedo demais, perde neutralidade e vira gerador de boilerplate.

## Confusao entre AST e IR

Sem separar representacao sintatica e semantica, qualquer evolucao futura vira sofrimento desnecessario.
